# B.I.O.M.A. — Editions (Open-Core)

**🌐 English · [Português](EDITIONS.pt-BR.md)**

B.I.O.M.A. follows an **open-core** model: the core is free and open (MIT); a separate
**Enterprise** edition, under a commercial license, adds the sovereignty, compliance and
support layer that regulated organizations need.

## Community Edition — this repository, MIT

Free and open. Use it, fork it, embed it, ship it — at no cost, under the [`LICENSE`](LICENSE).

- `bioma_micro` — the Rust micro-kernel: hormonal bus, context apoptosis, `saturation_scan`.
- `bioma` — `CognitiveFirewall` (secret redaction, cognitive-DDoS mitigation, timeout guard),
  provider-agnostic client, local server.
- Full test suite, benchmarks, and documentation.

**Everything the ground-truth benchmarks prove (−80–97% tokens, secret redaction,
cognitive-DDoS mitigation, μs kernel) is in the free Community edition.**

## Enterprise Edition — private repo `bioma-enterprise`, commercial license

Closed-source add-ons for **sovereign / air-gapped / regulated** deployments. Builds on the
MIT core; requires a commercial license.

| Capability | Community (MIT) | Enterprise (commercial) |
| :--- | :---: | :---: |
| Context apoptosis · firewall · kernel | ✅ | ✅ |
| Provider-agnostic (Anthropic/Google/OpenAI/local) | ✅ | ✅ |
| Sovereign / air-gapped deployment tooling & hardening | — | ✅ |
| Centralized policy & vault management (PII packs, org-wide secrets) | basic | ✅ |
| Admin dashboard & fleet observability | telemetry only | ✅ |
| Compliance evidence (audit logs, LGPD / SOC 2 / ISO 27001) | — | ✅ |
| Priority support & SLA | community | ✅ |

Enterprise is where the differentiation the market analysis identified lives — the
**sovereign, on-prem angle** that SaaS competitors cannot serve.

## Why open-core

The MIT core drives **adoption, trust and credibility**; the Enterprise edition **funds
development**. Using the free core never obligates you to Enterprise — Enterprise exists for
organizations that need the sovereignty/compliance/support layer on top.
