## Honest caveats

1. **N=1 per arm.** One execution each; turns vary between runs of the same
   arm (agent variance). Do not use the turn delta (24 vs 42) as a claim.
2. **Prompt-cache interaction.** The CLI's cost accounting favored arm A
   ($0.66 vs $6.62 notional) because pruning changes the prefix and
   invalidates the native cache when `stable_prefix` is minimal (2). The net
   post-cache gain with a properly sized stable zone is what the public
   benchmark measures (−42% vs cache; −71% net on the hot call) — this E2E
   deliberately did not optimize for it, to isolate apoptosis. Addressed in
   v1.4.0 (`BIOMA_PURGE_QUANTUM`, `BIOMA_CACHE_HYSTERESIS`) — see
   `BIOMA_VALIDATION_EVOLUTIONS.md`.
3. **Extra turns in arm B** are consistent with re-reads of files whose
   content was pruned — final quality intact, throughput tunable via
   threshold/half-life/stable_prefix.

## Environment findings (apply to ANY proxy, not just BIOMA)

1. **Claude Code CLI 2.1.x assumes a small window (~30k) with a third-party
   `ANTHROPIC_BASE_URL`** and enters "Autocompact is thrashing" around turn
   10. Fix: declare the capability on the model (`--model "sonnet[1m]"`).
   `CLAUDE_CODE_AUTO_COMPACT_WINDOW` had no effect on this version.
2. **Autocompact is client-side accounting** (local transcript): arms with
   and without server-side pruning fail identically. BIOMA's savings are on
   the wire and the bill; the CLI's local context management is unaffected.
3. **The native API is strict about `messages[0]`**: if pruning removes the
   first real turn, the upstream returns 400. `BIOMA_STABLE_PREFIX>=1` fixes
   it and also protects the cache prefix. *Product decision: the Anthropic
   surface now enforces a minimum of 1 stable unit (v1.4.0).*
4. **Subscription OAuth requires forwarding the `anthropic-beta` header** —
   fixed in the gateway on this date (OpenRouter-style upstreams ignore it).
